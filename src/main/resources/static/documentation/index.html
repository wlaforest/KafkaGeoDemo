A) HTML Files - top

Clean admin comes with 4 HTML files:

  • index.html - login form
  • register.html - register form
  • home.html - page with all HTML elements, including the menu
  • single-page.html - single/static page with menu and plain text content

Every file has a minimal set of included CSS and JS files in order to minimize load times. If you're having issues with any JS component please make sure you include all the necessary files and init the script as well.

B) HTML Structure - top

Template's layout is fixed, 960px wide with 1 main center column that can be divided in multiple sub-columns with a minimum width of 1/16. Template uses the 960 Grid System, HTML5 and CSS3.

When editing the HTML please keep in mind the structure below and try to use one of the provided templates. It'll make your job a lot easier. Layout wise, there's no need to edit the CSS. Everything can be accomplished with the predefined classes. Just look at the examples.

Making the main menu "normal" (non-sticky)

To make the menu scroll with the rest of the content (make it non-sticky) just remove "sticky" class from DIV with id "navigation".

Modifying the main menu

Main menu is located in navigation div. Each LI element in ul#main-nav represents one menu item. If you want to create submenus just create a sub UL element using the same rules. Apply a "current_menu_item" class to any LI element to mark it as the selected item. Simple menu example:


Removing the open/close button from boxes
If you want to remove the open/close button from a single box just remove the SPAN element with "show_hide" from that box. If you want to disable that functionality on the whole site open common.js and remove the JS from line #54 to #58.

C) CSS Files and Colors Themes - top

List of included color themes (10):

  • blue.css (default)
  • brown.css
  • golden.css
  • green.css
  • orange.css
  • pink.css
  • red.css
  • silver.css
  • silver-dark.css
  • teal.css

Each color theme has its own CSS file located in /css/color-themes/. Blue is the default theme, included in theme's header. To change the theme just include a different theme CSS file. Please include only one theme at a time.

List of included background textures (5):

  • clean.css
  • squares.css
  • stripes.css
  • stripes-rev.css
  • crosshatch.css (default)

Each background texture has its own CSS file located in /css/textures/. Crosshatch is the default background, included in theme's header. To change the background just include a different background CSS file. Please include only one background at a time.

Other CSS files

  • styles.css - main style file; the only one you should ever modify
  • 960.css - 960gs original file
  • font-awesome.css - font based icons
  • ie8.css - IE8 fixes
  • prettyPhoto.css - prettyPhoto lightbox styles; un-include this file if you're not using prettyPhoto

The main CSS file (style.css) is separated into following sections for easier editing:

If you would like to edit a specific template element, simply find the appropriate label in the CSS file, and then scroll down until you find the style that needs editing.

Changing fonts
Fonts are defined on the beginning of main CSS file. Default fonts are Google Fonts - PT Sans and PT Serif. You can easily change the font family by changing those 3 font definitions. Font-size and line-height can also be easily changed. Please bear in mind that changing those properties will significantly change the design and overall appearance of the template.

Using font based icons from Font Awesome
We've included more than 150 awesome font based icons (along with +500 image icons). In order touse font icons header over to Font awesome find the icon you need and copy paste it's class (ie: icon-book). Then create an empty SPAN element and just give it that class.

Please note: font icons DO NOT work on file:// protocol (ie: file preview). You have to view the template via http:// protocol.

Using message boxes
Clean Admin comes with five predefined message box styles givingyou an option to display; information, warning, error, success or simple message. You can use the messages anywhere in the template; they resize automatically.

Using the buttons
Clean Admin supports three buttons sizes: small, medium and large; as well as six button colors: silver, black, blue, green, orange, red giving you a total of 18 button combinations. Just apply two classes to your buttons as shown below to control the size and the color. You can, of course, use the buttons anywhere on the site.


Using the tabs, vertical and horizontal
You can easily create vertical and horizontal tabs with some simple markup. Tabs consist of two UL elements. First one holds the links/tabs list and the other one the content. First list has to have the "tabs" class and the second one "tabs-content" class. Ancors in links have to match the IDs of LI elements in the second list. Here's a simple example of horizontal tabs.

Converting horizontal tabs to vertical is easy. Just add "vertical-tabs" class to the first list, and "vertical-tab-content" class to the second one.

D) PSD Files - top

Clean Admin has a minimal amount of images in order to make the load time minimal. However, we've included two PSD files to make customizing easier. You can find the files in PSD folder. They're properly layered for easier editing.

  • cleanadmin-background.psd - header background
  • cleanadmin-logo.psd - easily editable logo with tagline

E) JavaScript - top

All JS code is jQuery based. Here's the list of JS files. They're located in the /js/ folder.

  • jquery.prettyPhoto.js - prettyPhoto lightbox script
  • jquery.tablesorter.js - tablesorter script
  • jquery.ui.custom.js - date and other UI components
  • jquery.metadata.js - metadata extraction helper
  • jquery.tipsy.js - tooltips script
  • jquery.flot.js - graphs plotting script
  • excanvas.js - IE fix for plot graphs
  • markitup/jquery.markitup.js - WYSIWYG editor
  • tabs.js - simple tabs script
  • plot-init.js - example graph init
  • common.js - misc code, script inits and config for all scripts


Configuring the scripts
All scripts are initialized in common.js file which is very short and heavily commented. If you don't want to use any of the included scripts be sure to remove them from HTML (remove their include line) and also remove the init code from common.js. For further info on any script please have a look at their original documentation. You can find the links at the bottom of this document.

  • converting input fields into "fancy date" fields is done by giving the field a "input-date" class
  • converting textareas into WYSIWYG editors is done by giving the textarea a "markitup" class
  • converting regular tables input sortable ones is done by giving the table a "datatable" class
